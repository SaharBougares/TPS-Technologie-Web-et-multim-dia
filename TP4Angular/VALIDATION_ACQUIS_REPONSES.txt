##  Question 1 : Fonctionnement de `*ngIf`

### Énoncé
Quelle est la principale fonctionnalité de la directive `*ngIf` dans Angular ?

A) Itérer sur une liste d'éléments
B) Appliquer des styles conditionnels
C) Contrôler l'affichage conditionnel des éléments
D) Lier des données bidirectionnellement

###  Réponse Correcte
**C) Contrôler l'affichage conditionnel des éléments**

### Explication
La directive `*ngIf` permet de contrôler l'affichage ou le masquage d'éléments DOM en fonction d'une condition booléenne. Elle ajoute ou supprime l'élément du DOM selon que la condition est vraie ou fausse.

### Implémentation dans le Projet
```html
<!-- app.html -->
<div *ngIf="isReponse; else incorrectMessage" class="feedback-message correct">
  <p> Bonne réponse ! Vous gagnez 10 points.</p>
</div>
<ng-template #incorrectMessage>
  <div *ngIf="showFeedback" class="feedback-message incorrect">
    <p>❌ Mauvaise réponse. Vous perdez 5 points.</p>
  </div>
</ng-template>

##  Question 2 : Fonctionnement de `*ngFor`

### Énoncé
Quelle directive Angular est utilisée pour itérer sur une collection et afficher chaque élément ?

A) `*ngIf`
B) `*ngFor`
C) `[ngClass]`
D) `[ngStyle]`

###  Réponse Correcte
**B) `*ngFor`**

### Explication
La directive `*ngFor` permet d'itérer sur une collection (tableau, liste) et de créer une instance du template pour chaque élément de la collection. Elle est utilisée pour afficher des listes dynamiques.

### Implémentation dans le Projet
```html
<!-- app.html -->
<div *ngFor="let question of questions">
  <div *ngIf="!question.answered">
    <h3>{{ question.question }}</h3>
    

    <button *ngFor="let option of question.options"
      (click)="onSelectOption(option, question)">
      {{ option }}
    </button>
  </div>
</div>

##  Question 3 : Application de `*ngIf`

### Énoncé
Comment la directive `*ngIf` contribue-t-elle à la réactivité d'une application Angular ?

A) En permettant de modifier les styles d'un élément
B) En contrôlant dynamiquement l'affichage des éléments en fonction des données
C) En facilitant la liaison bidirectionnelle des données
D) En itérant sur des collections de données

###  Réponse Correcte
**B) En contrôlant dynamiquement l'affichage des éléments en fonction des données**

### Explication
La réactivité d'une application Angular dépend de sa capacité à mettre à jour l'interface utilisateur en fonction des changements de données. `*ngIf` contribue à cette réactivité en affichant ou masquant dynamiquement des éléments selon l'état des données.

### Implémentation dans le Projet
```html
<!-- app.html --->
<div *ngIf="!question.answered">
  <!-- Affiche la question uniquement si elle n'a pas été répondue -->
  <h3>{{ question.question }}</h3>
</div>

<!-- app.html  -->
<button *ngIf="allQuestionsAnswered" (click)="finishGame()" class="finish-button">
  ✓ Terminer le jeu
</button>
```

**Réactivité démontrée:**
- Questions masquées dynamiquement après réponse
- Bouton "Terminer" apparaît automatiquement quand toutes les questions sont répondues



##  Question 4 : Utilisation de `[ngClass]`

### Énoncé
Comment `[ngClass]` améliore-t-il la gestion des styles dans une application Angular ?

A) En itérant sur des listes d'éléments
B) En appliquant des classes CSS de manière conditionnelle
C) En liant des données bidirectionnellement
D) En contrôlant l'affichage des éléments

###  Réponse Correcte
**B) En appliquant des classes CSS de manière conditionnelle**

### Explication
`[ngClass]` permet d'appliquer ou de retirer des classes CSS dynamiquement en fonction de conditions. Cela améliore la gestion des styles en permettant des changements visuels réactifs sans modifier le DOM.

### Implémentation dans le Projet
```html
<!-- app.html -->
<button 
  *ngFor="let option of question.options"
  class="quiz-button"
  [ngClass]="{
    'correct-answer': question.answered && isCorrect(option, question),
    'incorrect-answer': question.answered && !isCorrect(option, question) && option === question.selectedAnswer,
    'neutral-answer': question.answered && !isCorrect(option, question) && option !== question.selectedAnswer
  }"
  (click)="onSelectOption(option, question)">
  {{ option }}
</button>
```

### Classes CSS Définies
```scss
/* app.scss - */
.correct-answer {
  background: #22c55e !important;  /* VERT */
  color: white !important;
}

/* app.scss -  */
.incorrect-answer {
  background: #ef4444 !important;  /* ROUGE */
  color: white !important;
}

##  Question 5 : Utilisation de `[ngStyle]`

### Énoncé
Dans quel scénario préféreriez-vous utiliser `[ngStyle]` plutôt que `[ngClass]` ?

A) Lorsque vous souhaitez itérer sur une liste d'éléments
B) Lorsque vous avez besoin d'appliquer plusieurs classes conditionnellement
C) Lorsque vous devez appliquer des styles CSS spécifiques en fonction des données
D) Lorsque vous voulez contrôler l'affichage d'un élément

###  Réponse Correcte
**C) Lorsque vous devez appliquer des styles CSS spécifiques en fonction des données**

### Explication
`[ngStyle]` est préféré quand vous avez besoin d'appliquer des styles CSS spécifiques (comme la couleur, la taille, l'opacité) directement en fonction des données, plutôt que d'utiliser des classes CSS prédéfinies.

### Implémentation dans le Projet
```html
<!-- app.html  -->
<button 
  *ngFor="let option of question.options"
  class="quiz-button"
  [ngClass]="{...}"
  [disabled]="question.answered"
  (click)="onSelectOption(option, question)">
  {{ option }}
</button>
```

**Alternative avec `[ngStyle]`:**
```html
[ngStyle]="{'opacity': question.answered ? '0.5' : '1'}"
```

**utilisation de  `[disabled]` pour un effet similaire**



##  Question 6 : Combinaison de Directives

### Énoncé
Quel est l'avantage principal de combiner `*ngIf` et `*ngFor` dans une application Angular ?

A) Améliorer la gestion des événements utilisateur
B) Contrôler l'affichage conditionnel tout en itérant sur des collections de données
C) Appliquer des styles dynamiques
D) Faciliter la liaison bidirectionnelle des données

###  Réponse Correcte
**B) Contrôler l'affichage conditionnel tout en itérant sur des collections de données**

### Explication
Combiner `*ngIf` et `*ngFor` permet de créer des interfaces complexes où vous itérez sur une collection ET contrôlez l'affichage de chaque élément selon des conditions spécifiques.

### Implémentation dans le Projet
```html
<!-- app.html - -->
<div *ngFor="let question of questions">
  <!-- *ngFor itère sur les questions -->
  
  <div *ngIf="!question.answered">
    <!-- *ngIf contrôle l'affichage selon l'état de la question -->
    
    <h3>{{ question.question }}</h3>
    
    <button *ngFor="let option of question.options"
      <!-- *ngFor imbriqué pour itérer sur les options -->
      (click)="onSelectOption(option, question)">
      {{ option }}
    </button>
  </div>
</div>
```

**Avantages démontrés:**
- Itération sur les questions (`*ngFor`)
- Affichage conditionnel selon l'état (`*ngIf`)
- Itération imbriquée sur les options (`*ngFor`)

##  Question 7 : Impact sur la Réactivité

### Énoncé
Comment l'utilisation combinée de `*ngIf`, `*ngFor`, `[ngClass]` et `[ngStyle]` affecte-t-elle la réactivité de l'application ?

A) Elle ralentit l'application en augmentant le nombre de directives
B) Elle améliore la réactivité en permettant des mises à jour dynamiques et conditionnelles de l'interface utilisateur
C) Elle n'a aucun impact significatif sur la réactivité
D) Elle complique le code sans bénéfices réels

###  Réponse Correcte
**B) Elle améliore la réactivité en permettant des mises à jour dynamiques et conditionnelles de l'interface utilisateur**

### Explication
Ces directives permettent à l'interface utilisateur de se mettre à jour automatiquement et dynamiquement en fonction des changements de données, ce qui est la définition même de la réactivité dans Angular.

### Implémentation dans le Projet

**Réactivité démontrée:**

1. **Affichage dynamique des questions:**
```html
<div *ngIf="!question.answered">
  <!-- Mise à jour automatique quand question.answered change -->
</div>
```

2. **Couleurs dynamiques des boutons:**
```html
[ngClass]="{
  'correct-answer': question.answered && isCorrect(option, question),
  'incorrect-answer': question.answered && !isCorrect(option, question) && option === question.selectedAnswer
}"
```

3. **Historique dynamique:**
```html
<div *ngFor="let item of answerHistory">
  <!-- Mise à jour automatique quand answerHistory change -->
</div>
```

4. **Bouton "Terminer" dynamique:**
```html
<button *ngIf="allQuestionsAnswered">
  <!-- Apparaît automatiquement quand toutes les questions sont répondues -->
</button>
```


##  Question 8 : Gestion des Classes Dynamiques

### Énoncé
Quelle directive permet d'appliquer une classe CSS spécifique à un élément uniquement si une condition est remplie ?

A) `*ngIf`
B) `*ngFor`
C) `[ngClass]`
D) `[ngStyle]`

###  Réponse Correcte
**C) `[ngClass]`**

### Explication
`[ngClass]` permet d'appliquer une ou plusieurs classes CSS de manière conditionnelle. Vous pouvez spécifier une condition pour chaque classe, et elle ne sera appliquée que si la condition est vraie.

### Implémentation dans le  Projet
```html
<!-- app.html  -->
[ngClass]="{
  'correct-answer': question.answered && isCorrect(option, question),
  'incorrect-answer': question.answered && !isCorrect(option, question) && option === question.selectedAnswer,
  'neutral-answer': question.answered && !isCorrect(option, question) && option !== question.selectedAnswer
}"
```

**Classes appliquées conditionnellement:**
- `correct-answer` si la condition est vraie
- `incorrect-answer` si la condition est vraie
- `neutral-answer` si la condition est vraie

##  Question 9 : Modification des Styles Dynamique

### Énoncé
Quelle directive Angular vous permet de modifier dynamiquement les propriétés CSS telles que la couleur, la taille ou l'opacité d'un élément ?

A) `*ngIf`
B) `*ngFor`
C) `[ngClass]`
D) `[ngStyle]`

###  Réponse Correcte
**D) `[ngStyle]`**

### Explication
`[ngStyle]` permet de modifier directement les propriétés CSS d'un élément en fonction des données. Vous pouvez contrôler la couleur, la taille, l'opacité, et toute autre propriété CSS dynamiquement.

### Implémentation dans Votre Projet
```html
<!-- app.html -->
<button 
  [disabled]="question.answered"
  (click)="onSelectOption(option, question)">
  {{ option }}
</button>
```

**Alternative avec `[ngStyle]`:**
```html
[ngStyle]="{
  'opacity': question.answered ? '0.5' : '1',
  'cursor': question.answered ? 'not-allowed' : 'pointer'
}"
```


##  Question 10 : Création d'une Application Complète

### Énoncé
Quelle approche est la plus appropriée pour intégrer efficacement les directives `*ngIf`, `*ngFor`, `[ngClass]` et `[ngStyle]` dans une application complète de gestion des étudiants afin d'améliorer l'interactivité et l'expérience utilisateur ?

A) Utiliser uniquement `*ngFor` pour toutes les itérations et ignorer les autres directives
B) Combiner `*ngIf` et `*ngFor` pour contrôler l'affichage et l'itération, tout en utilisant `[ngClass]` et `[ngStyle]` pour appliquer des styles conditionnels en fonction des données des étudiants
C) Appliquer des styles statiques dans les templates sans utiliser les directives Angular
D) Utiliser `*ngIf` pour tout le contrôle d'affichage sans combiner avec d'autres directives

###  Réponse Correcte
**B) Combiner `*ngIf` et `*ngFor` pour contrôler l'affichage et l'itération, tout en utilisant `[ngClass]` et `[ngStyle]` pour appliquer des styles conditionnels en fonction des données des étudiants**

### Explication
L'approche la plus efficace est de combiner toutes les directives de manière appropriée :
- `*ngFor` pour itérer sur les collections
- `*ngIf` pour contrôler l'affichage conditionnel
- `[ngClass]` pour appliquer des classes CSS conditionnellement
- `[ngStyle]` pour modifier les styles CSS dynamiquement

### Implémentation dans le Projet


1. **`*ngFor` pour l'itération:**
```html
<div *ngFor="let question of questions">
  <button *ngFor="let option of question.options">
  <div *ngFor="let item of answerHistory">
```

2. **`*ngIf` pour le contrôle d'affichage:**
```html
<div *ngIf="!question.answered">
<button *ngIf="allQuestionsAnswered">
<div *ngIf="answerHistory.length > 0">
```

3. **`[ngClass]` pour les styles conditionnels:**
```html
[ngClass]="{
  'correct-answer': question.answered && isCorrect(option, question),
  'incorrect-answer': question.answered && !isCorrect(option, question) && option === question.selectedAnswer
}"
```

4. **`[disabled]` pour les styles dynamiques:**
```html
[disabled]="question.answered"
```
